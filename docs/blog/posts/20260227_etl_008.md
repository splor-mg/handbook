## Discussão

### Atividade realizada

Foi mostrado no [repositório do painel-ldo](https://github.com/splor-mg/painel-ldo), o workflow que é utilizado para atualizar o [painel](https://painel-ldo-2027.streamlit.app/) sempre que ocorrer um commit na main.

Esse processo:

    1. define variáveis de ambiente

    2. cria uma máquina na nuvem

    3. clona o repositório

    4. instala o R, Python e Poetry

    5. roda a `task painel`

    6. faz commit e push dos dados

Além disso, localmente, foi executado o `streamlit run app.py` para visualizar o painel gerado.


#### Observações

1. Foi criada a `task toml` que cria o arquivo "data.toml" dinamicamente.

    !!! tip "Dica"
        Para mais detalhes, veja o [issue](https://github.com/splor-mg/painel-ldo/issues/2).

2. Ao longo da reunião, é mostrado como fazer as instalações necessárias para executar as tasks do [repositório painel-ldo](https://github.com/splor-mg/painel-ldo), inclusive mostra como conseguir o token no GitHub, para acessar repositórios privados.

3. Foi comentado ainda sobre o uso da "[frictionless](https://framework.frictionlessdata.io/)" para ler os recursos desejados e somente após utilizar o "[pandas](https://pandas.pydata.org/)" para transformar em data.frame e fazer as transformações necessárias.


### Resultados

Foi adicionado mais uma coluna com os dados do ano anterior ao de referência.
